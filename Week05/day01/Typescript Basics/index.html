Introduction to TypeScript

Typescript is an open source programming language which is mantained by Microsoft. It is a superset of Javascript and has more features than javascript.


Why use TypeScript?

It is a fully Object oriented Programming language. It has no Interpreter and is fully compiled language. It is a strongly typed programming language. In typescript, interface plays an important role.


Difference between TypeScript and JavaScript

JavaScript TypeScript
It is both scripting and programming language. It is a programming language.
It is both interpreting and compiling language. It is a compiling language. There is no Interpreter.
It is not fully object oriented programming language. It is fully Object Oriented Programming language.
There is no Interface. Interfaces play an important role in TypeScript.
It is not a strongly typed programming language. It is a strongly typed programming language.

DataTypes in TypeScript

Datatypes in Typescript are categorized into three major types:
  • Built-In DataTypes
  • User Defined Datatypes
  • Any DataType

Built-In DataTypes in TypeScript

These are pre-defined datatypes available in typescript. Look at the table mentioned below.
DataType Name Description of the DataType
Number DataType You can assign any numeric values including decimals, hexadecimals, binary, and octal literals. 
                // Number DataType
                var a: number;
                a = 234;
                console.log("Value of a : ", a);
String DataType Textual Data is stored in String.
TypeScript also uses double quotes (") and single quotes (') to surround the string value. 
                // String DataType
                var b: string;
                b = "Welcome to Typescript";
                console.log("Value of b : ", b);
Boolean DataType True/False Values
                // Boolean DataType
                var c: boolean;
                c = false;
                console.log("Value of c : ", c);
Void DataType It means Nothing.
It can only be assigned undefined or null to them.
                // Void DataType
                var d: void;
                console.log("Value of d : ", d);
Null DataType It can be assigned only null,number or boolean value to it.
                // Null DataType
                var e: null;
                e = null;
                console.log("Value of e : ", e);
Undefined DataType You can assign it to a number or a boolean value. 
                // Undefined Datatype
                var f: undefined;
                console.log("Value of f : ", f);
                var g: number;
                console.log("Value of g : ", g);

User Defined DataTypes in TypeScript

1. Array DataType

An array is a collection of elements of the same data type. 
// Array DataTypes
var arr1: number[];
arr1 = [1, 2, 3, 4, 5];
console.log("Array is : ", arr1);
console.log("Type of arr1 is : ", typeof arr1);

2. Class DataType

Classes are used to create reusable components and acts as a template for creating objects. 
// Class DataType
class Person {
  firstName;
  lastName;

  setName(f: string, l: string) {
    this.firstName = f;
    this.lastName = l;
  }
  getName() {
    return "Your Name is : " + this.firstName + " " + this.lastName;
  }
}
var p1 = new Person();
p1.setName("Raman", "Mishra");
console.log(p1.getName());

3. Interface DataType

In general, it defines the specifications of an entity. It has no implementation and only declarations. Classes are used to implement data members of the interface. 
class UserClass implements UserInterface {
  name;
  age;
  setData(n: string, a: number) {
    this.name = n;
    this.age = a;
  }
  displayName() {
    console.log("Your Name is : ", this.name);
  }
  checkVoteStatus(): string {
    if (this.age > 18) return "Yay! You can vote!";
    return "Oh No! You can't vote right now!";
  }
}
var u1 = new UserClass();
u1.setData("Minni", 34);
u1.displayName();
console.log(u1.checkVoteStatus());

4. Enum DataType

TypeScript provides both string-based and numeric-based enums. By default, enums begin numbering their elements starting from 0, but we can also change this by manually setting the value to one of its elements. 
// Enum DataType
enum MediaType {
  magazine,
  newspaper = 67,
  television,
  social_media,
}
function how2spread(projectName: string): MediaType {
  if (projectName == "instagram") return MediaType.social_media;
  return MediaType.magazine;
}
console.log(how2spread("instagram"));
console.log(how2spread("facebook"));

Any DataType

Any Type represents the super type of all data types in TypeScript. It helps to achieve optional dynamic datatype. 
// Any DataType
var h: any;
console.log("Value of h : ", h);
console.log("Type of h : ", typeof h);
h = 1234;
console.log("Value of h : ", h);
console.log("Type of h : ", typeof h);

console.log("Value of h : ", h);

console.log("Value of h : ", h);
console.log("Type of h : ", typeof h);
h = 45.3;
console.log("Value of h : ", h);
console.log("Type of h : ", typeof h);
h = null;
console.log("Value of h : ", h);
console.log("Type of h : ", typeof h);


Union of DataTypes

Union types are a powerful way to express a value that can be one of the several types. Two or more data types are combined using the pipe symbol (|) to denote a Union Type. 
// Union of DataTypes
var i: number | string;
console.log("Value of i : ", i);
console.log("Type of i : ", typeof i);
i = 1244;
console.log("Value of i : ", i);
console.log("Type of i : ", typeof i);
i = "hello";
console.log("Value of i : ", i);
console.log("Type of i : ", typeof i);

Intersection of Data Types

An intersection type is a type that merges several kinds into one. This allows you to combine many types to create a single type with all of the properties that you require. The ‘&’ operator is used to create the intersection type. 
// Intersection of DataTypes
interface Student {
  student_id: number;
  name: string;
}

interface Teacher {
  Teacher_Id: number;
  teacher_name: string;
}

type intersected_type = Student & Teacher;

var obj1: intersected_type = {
  student_id: 3232,
  name: "rita",
  Teacher_Id: 7873,
  teacher_name: "seema",
};

console.log(obj1.Teacher_Id);
console.log(obj1.name);
