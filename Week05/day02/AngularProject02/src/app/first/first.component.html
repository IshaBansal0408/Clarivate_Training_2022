Data Binding Introduction

Data Binding refers to exchange of information between the View file and the Model file. Whenever the user updates the data in the view, Angular updates the component. When the component gets new data, the Angular updates the view. There are two types of Data Binding: One-Way and Two-Way.

One-Way Data Binding

In 'one way binding'data flows from one direction. Either from view to component or from component to view.

1. Interpolation

The Angular uses the double curly braces in the template to denote the interpolation. The content inside the double braces is called 'Template Expression'.

Here is the Interpolation data binding variable => {{ variable01 }}

The Angular first evaluates the Template Expression and converts it into a string. Then it replaces Template expression with the result in the original string in the HTML. Whenever the template expression changes, the Angular updates the original string again

5. Event Binding

Event binding allows us to bind events such as keystrokes, clicks, hover, touch, etc to a method in component. It is one way from view to component.

Two-Way Data Binding

'Two-way binding' means that changes made to our model in the component are propagated to the view and that any changes made in the view are immediately updated in the underlying component.
The two-way binding uses the special syntax known as a banana in a box [()].
The Angular uses the 'ngModel' directive to achieve the two-way binding on HTML Form elements.
